# TrustExam — Система онлайн-прокторинга

Платформа для проведения онлайн-экзаменов с AI-прокторингом, записью экрана и камеры.

---

## Структура проекта

```
trustexam/
├── server.js                  # Backend: Express + Socket.IO
├── recordings/                # Папка с видеозаписями (создаётся автоматически)
│
└── src/
    ├── main.jsx               # Точка входа React
    ├── App.jsx                # Роутинг: Login → Student / Proctor
    ├── socket.js              # Singleton Socket.IO клиент
    │
    ├── StudentExam.jsx        # Интерфейс студента (setup → exam → done)
    ├── ProctorDashboard.jsx   # Панель проктора (студенты + записи)
    │
    ├── useRecording.js        # Hook: раздельная запись экрана и камеры
    └── useProctoring.js       # Hook: AI-детекция лица, поворота головы
```

---

## Компоненты и их функции

### `App.jsx`
**Роль:** корневой роутер  
- Хранит `session` (role, username, studentId)  
- Рендерит `LoginPage`, `StudentExam` или `ProctorDashboard`  
- `LoginPage` — форма входа с выбором роли (студент / проктор)

---

### `StudentExam.jsx`
**Роль:** весь интерфейс студента во время экзамена

| Фаза | Что происходит |
|------|---------------|
| `setup` | Включение камеры, подтверждение одиночества, старт записи |
| `exam` | Вопросы с вариантами ответов, AI-прокторинг активен |
| `done` | Результат автопроверки + оценка проктора |
| `banned` | Блокировка при 5 нарушениях |

**Ключевые функции:**
- `handleCamera()` — запрашивает доступ к камере через `useRecording`
- `startExam()` — запускает запись экрана + камеры, fullscreen, AI-детекцию
- `submitExam()` — останавливает запись, выходит из fullscreen, отправляет ответы
- `selectAnswer()` — выбор ответа (заблокирован если `isBlocked === true`)
- `handleBanned()` — останавливает всё при получении события бана

**Слушает Socket события:**
- `students_update` — обновление нарушений, оценки, статуса
- `banned` — немедленная блокировка от проктора

---

### `ProctorDashboard.jsx`
**Роль:** панель наблюдения проктора

**Таб "Студенты":**
- Список всех студентов с цветовыми статусами (онлайн / предупреждён / забанен / оффлайн)
- При клике — детальная панель: журнал нарушений, оценка, кнопки действий
- `setVerdict('banned')` — баниит студента (шлёт socket `banned` студенту)
- `setVerdict('cleared')` — снимает статус
- `submitScore()` — выставляет оценку проктора
- `sendComment()` — добавляет комментарий в журнал

**Таб "Записи":**
- Сетка всех записанных сессий
- Для каждого студента — **две кнопки**: "Экран" и "Лицо" (раздельные файлы)
- Видеоплеер в модальном окне
- Скачивание обоих файлов

---

### `useRecording.js`
**Роль:** управление записью — РАЗДЕЛЬНО экран и камера

**Что записывает:**
- `{hash}_screen.webm` — запись экрана в высоком качестве (1280×720, 3 Mbps)
- `{hash}_face.webm` — запись лица в высоком качестве (640×480, 1.5 Mbps)

**Публичные методы:**
| Метод | Описание |
|-------|----------|
| `requestCamera()` | Запрашивает доступ к камере, возвращает stream |
| `startRecording(cameraStream)` | Запускает шейринг экрана + два MediaRecorder |
| `stopRecording()` | Останавливает оба recorder, флашит чанки, закрывает файлы |

**Как работает:**
1. `startRecording` → запрашивает `getDisplayMedia` (экран)
2. Создаёт два `MediaRecorder`: один на `screenStream`, другой на `cameraStream`
3. Каждые 5 секунд флашит чанки на сервер с заголовком `x-recording-type: screen` или `face`
4. `stopRecording` → флашит остаток, останавливает все треки

---

### `useProctoring.js`
**Роль:** AI-детекция нарушений через TensorFlow.js

**Что детектирует:**
| Нарушение | Порог | Действие |
|-----------|-------|----------|
| Лицо не в кадре | 3 секунды | violation + блокировка ответов |
| Поворот головы | 0.5с блок / 2.5с violation | блокировка + отчёт |
| Второй человек | мгновенно | critical violation |

**Публичные методы:**
| Метод | Описание |
|-------|----------|
| `initAI(stream)` | Загружает TF.js, COCO-SSD, FaceMesh модели |
| `startDetectionLoop()` | Запускает цикл детекции каждые 2 секунды |
| `stopDetection()` | Останавливает цикл, очищает ресурсы |
| `attachStream(stream)` | Привязывает видеопоток к offscreen video элементу |

**Возвращает:**
- `warning` — текст предупреждения для overlay (null если всё ок)
- `isBlocked` — блокирует выбор ответов
- `status` / `statusText` — состояние AI системы

---

### `server.js`
**Роль:** backend сервер

**REST API:**

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/students` | Список всех студентов |
| POST | `/api/students/:id/comment` | Добавить комментарий проктора |
| POST | `/api/students/:id/verdict` | Установить вердикт (banned/cleared) |
| POST | `/api/students/:id/score` | Выставить оценку проктора |
| POST | `/api/students/:id/submit-answers` | Автоподсчёт результата |
| POST | `/api/recording/start` | Начать запись, получить hash |
| POST | `/api/recording/chunk` | Принять чанк видео (screen или face) |
| POST | `/api/recording/stop` | Завершить запись, закрыть файлы |
| GET | `/api/recordings` | Список всех записей |
| GET | `/recordings/:file` | Отдать видеофайл (с Range support) |

**Socket.IO события:**

| Событие | От кого | Описание |
|---------|---------|----------|
| `student_join` | студент | регистрация в системе |
| `proctor_join` | проктор | подключение к панели |
| `ai_violation` | студент | отчёт о нарушении от AI |
| `students_update` | сервер → все | обновление списка студентов |
| `banned` | сервер → студент | принудительный бан |

## Запуск

## Правила автоподсчёта

Правильные ответы (индексы, начиная с 0):

| Вопрос | Правильный ответ |
|--------|-----------------|
| 1. Сложность O(n log n) | Быстрая сортировка (индекс 1) |
| 2. REST API | Архитектурный стиль (индекс 1) |
| 3. LIFO | Стек (индекс 2) |
| 4. SQL | Structured Query Language (индекс 0) |
| 5. HTTP создание | POST (индекс 2) |

## Система нарушений

```
0–2 нарушения  → статус "active"  (зелёный)
3–4 нарушения  → статус "warned"  (жёлтый)
5+  нарушений  → статус "banned"  (красный) + мгновенная блокировка студента
```

## Файлы записей

Каждая сессия создаёт **два файла**:
```
recordings/
├── a1b2c3d4e5f6_screen.webm   # Запись экрана (высокое качество)
└── a1b2c3d4e5f6_face.webm     # Запись лица   (высокое качество)
```

Hash генерируется из `studentId + timestamp + random` через SHA-256 (первые 16 символов).
